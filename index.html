SASS to CSS

CSS with Super Powers

Syntactically Awesome Style Sheets(SASS) is a preprocessor language that’s compiled to CSS. It allows you to use variables, nested rules, mixins, functions, imports and more, all with a fully CSS-compatible syntax. Sass helps keep large stylesheets well-organized and makes it easy to share design within and across projects. Sass is completely compatible with all versions of CSS. We take this compatibility seriously, so that you can seamlessly use any available CSS libraries. Sass is actively supported and developed by a consortium of several tech companies and hundreds of developers.

Features : CSS Compatible Feature Rich Large Community
Variables
Sass variables are simple: you assign a value to a name that begins with $, and then you can refer to that name instead of the value itself. Variables make it possible to reduce repetition. Sass variables, like all Sass identifiers, treat hyphens and underscores as identical. This means that $font-size and $font_size both refer to the same variable. 
 /* Variable declaration */
$blue:    #0d6efd; 
$indigo:  #6610f2;   

/* Key value pair variables */
$colors: (
"blue":     $blue,
"indigo":   $indigo,
)
/* Normal variable usage */
.text {
    color: $white; 
}
/* Using key value pair */
.content {
    color: map-get($colors, indigo);
    background-color: $white;
}
Function
Functions make it easy to abstract out common formulas and behaviors in a readable way. Functions are defined using the @function at-rule, which is written @function name(arguments...) { ... }. 
/* Function without arguments */
@function calc-fontSize() {
    @return 12px * $val;
}
/* Function with argument */
@function calc-width($val) {
    @return 100%/12 * $val;
}
/* Function with default arguments */
@function calc-fontSize($val:1) {
    @return 12px * $val;
}
/* Function without arguments */
#container-wrap {
    width: calc-width();
}
/* Function with argument */
#col-one {
    width: calc-width(1);
}
/* Function with default arguments */
#content {
    font-size: calc-fontSize(2);
}
Mixins
Mixins allow you to define styles that can be re-used throughout your stylesheet. They make it easy to distribute collections of styles in libraries. Mixins are defined using the @mixin at-rule, which is written @mixin name { ... } or @mixin name() { ... }. Mixins are included into the current context using the @include at-rule, which is written @include or @include (), with the name of the mixin being included. 
/* Plain Mixins */
@mixin heading {
    font-size: 18px;
    color: $indigo;
}
/* Mixin with input parameter */
@mixin align($val) {
    text-align: $val;
}
/* Mixin with default parameter */
@mixin article($fontSize:50px) {
    font-size: $fontSize;
    color: $orange;
}
/* Mixin with 'n' arguments */
@mixin transistion($param...) {
    transition: $param;
}

/* Plain Mixins */
#intro-heading {
    @include heading;
}
/* Mixin with input parameter */
#heading {
    color:$white;
    @include align(center);
}
/* Mixin with default parameter */
#content {
    @include article;
}
/* Mixin with 'n' arguments */
#content-wrap {
    color: $indigo;
    @include transistion(0.5s ease-in);
}
Extend
There are often cases when designing a page when one class should have all the styles of another class, as well as its own specific styles. Sometimes you want to write a style rule that’s only intended to be extended. In that case, you can use placeholder selectors, which look like class selectors that start with % instead of (.) 
.error {
    font-size: 18px;
    color: red;
}

/* extending another selector styles */
.service-error h1 {
    @extend .error; 
}

/* Placeholder it wont be compiled in css */
%hover {
    background-color: black;
    cursor: pointer;
}

Loops
 
                

                

                    

                        

                            Sass variables are simple: you assign a value to a name that begins with $, 
                            and then you can refer to that name instead of the value itself. Variables make it possible to reduce repetition.
                            Sass variables, like all Sass identifiers, treat hyphens and underscores as identical. 
                            This means that $font-size and $font_size both refer to the same variable.
                        

                        


/* For Loop */
@for $i from 1 to 12 {
    .col-#{$i} {
        width: calc-width($i);
        transition: 0.5s;
        &:hover {
            @extend %hover;
        }
    }
}
/* For each */
@each $name, $color in $colors {
    .#{$name}-bg {
        background-color:$color;
    }
}
/* While loop */
$x:6;
$y:1;
@while $x > 0 {
    h#{$x} {
        font-size: calc-fontSize($y);
        margin: .5rem;
    }
    $x : $x - 1;
    $y : $y + 1;
}


                        

                        

                            

                        

                    

                

            

        

        
        

            

                

                    
Maps
 
                

                

                    

                        

                            Sass variables are simple: you assign a value to a name that begins with $, 
                            and then you can refer to that name instead of the value itself. Variables make it possible to reduce repetition.
                            Sass variables, like all Sass identifiers, treat hyphens and underscores as identical. 
                            This means that $font-size and $font_size both refer to the same variable.
                        

                        


/*
map-get(map, key)       Returns value for specified key in map
map-has-key(map, key)   Checks whether map has specified key
map-keys(map)           Returns a list of all keys in map
map-values(map)         Returns a list of all values in map
map-merge(map1, map2)   Appends map2 to the end of map
map-remove(map, keys)   Removes the specified keys from map
*/

$font-size:(sm:12px, md:18px, lg:24px);
$small-size:(xxs:8px, xs:10px);

$full-size: map-merge($font-size, $small-size);
$screen : map-keys($font-size);
$screen-size : map-values($font-size);

p {
    @if map-has-key($font-size, md) {
        font-size: map-get($font-size, md);
    }
    @else {
        font-size: map-get($font-size, sm);
    }    
}


                    

                        

                            

                        

                    

                

            

        

        
        

            

                

                    
Imports
 
                

                

                    

                        

                            Sass extends CSS's @import rule with the ability to import Sass and CSS stylesheets, 
                            providing access to mixins, functions, and variables and combining multiple stylesheets' CSS together.
                            Unlike plain CSS imports, which require the browser to make multiple HTTP requests as it renders your page,
                            Sass imports are handled entirely during compilation.
                        

                        


/* importing sass files */
@import 'variables';
@import 'functions';
@import 'mixins';
@import 'loops';
@import 'map';
@import 'nested';
@import 'extend';
